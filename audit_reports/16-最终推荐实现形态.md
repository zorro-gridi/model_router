## 16.1 推荐系统形态

设计文档定义了 5 步推荐系统形态，逐项对比：

| #   | 设计要求                      | 实现状态  | 证据                                                                                          |
| --- | ----------------------------- | --------- | --------------------------------------------------------------------------------------------- |

**系统形态对齐度**: 第 1、2（核心）、5 步完全实现；第 3 步降级实现；第 4 步完全缺失。

**实际状态**: ⚠️ 部分删除

| 检查点                         | 状态      | 证据                                                                   |
| ------------------------------ | --------- | ---------------------------------------------------------------------- |
| Stage 不参与路由决策           | ✅ 已删除 | `decision_engine.py` 路由逻辑仅使用 `task_pattern` + `task_complexity` |
| `stage_detector.py` 文件仍存在 | ⚠️ 残留   | 1201 行文件仍在 `hooks/model_router/` 下                               |
| proxy.py 仍导入 stage_detector | ⚠️ 残留   | `proxy.py:1015`: `from stage_detector import _update_state_index`      |
| LLM 分类器仍输出 stage 维度    | ⚠️ 残留   | `llm_classifier.py` system prompt 定义 9 个 stage                      |
| Stage 关键词配置仍在维护       | ⚠️ 残留   | `stage_config.py` 中 PATTERN_CONFIG 仍含 stage→pattern 映射            |

**评估**: Stage 路由逻辑已从核心决策路径移除（✅），但 `stage_detector.py`（1201 行）、LLM 分类器 stage 输出、stage_config 中 stage 关键词等辅助设施仍在运行/维护。属于"功能已拆，骨架仍在"的状态。

## 总体评估

| 子章节            | 对齐度    | 关键差异                                     |
| ----------------- | --------- | -------------------------------------------- |
| 16.1 推荐系统形态 | 70%       | 第 4 步缺失，第 2/3 步降级                   |
| 16.2 推荐保留项   | 67% (4/6) | Context Summary 缺失，TodoWrite/Runtime 降级 |
| 16.3 推荐删除项   | 75% (3/4) | Stage 骨架残留，YAML 未清理                  |

**综合评分**: 71% 对齐

**关键差异汇总**:

1. **Context Summary Injector 完全缺失** — 6 个推荐保留项中唯一零实现的项目
2. **Stage Router 骨架残留** — `stage_detector.py`（1201 行）仍在运行，LLM 分类器仍输出 stage，配置文件仍维护 stage 关键词。路由逻辑已移除但辅助设施未清理
3. **YAML 配置断电未拆线** — `config/decision_weights.yaml` + loader 函数存在但运行时不用
4. **降级逻辑 + 反复切换均成功删除** — 这两个推荐删除项完全对齐

**总结**: V1.3 实现正确地保留了核心路由维度（Pattern + Complexity + Lock），正确地删除了降级和反复切换。主要差距在于：Context Summary 完全缺失、Stage 和 YAML 辅助代码未清理（残留而非功能回归）。
